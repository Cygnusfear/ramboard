# 08.01 Visual System

> **Feature Area:** [[../README|08-Visual-Design]]
> **Feature ID:** 08.01

## Color System

### Base Palette (Dark theme only)
- **Background:** Zinc-950 (`#09090b`)
- **Surface:** Zinc-900 (`#18181b`)
- **Border:** Zinc-800 (`#27272a`)
- **Text primary:** Zinc-100 (`#f4f4f5`)
- **Text secondary:** Zinc-400 (`#a1a1aa`)
- **Text muted:** Zinc-500 (`#71717a`)

### Accent
- **Primary:** Blue-500 (`#3b82f6`) — selection, active states, primary buttons
- No purple. No neon. No gradients.

### Status Colors
- Open: Emerald-500 (`#10b981`)
- In Progress: Amber-500 (`#f59e0b`)
- Closed: Zinc-500 (`#71717a`)
- Cancelled: Red-500 (`#ef4444`)

### Priority Colors
- Urgent (0): Red-500
- High (1): Orange-500
- Medium (2): Blue-500
- Low (3): Zinc-500

### Tag Pills
Muted, desaturated backgrounds (`bg-{color}-500/10`) with matching text (`text-{color}-400`). Rounded-full, `px-2 py-0.5 text-xs`.

## Typography

- **Font stack:** Geist (sans), Geist Mono (code/IDs)
- **Headlines:** `text-lg font-medium tracking-tight`
- **Body:** `text-sm text-zinc-300`
- **Mono:** `text-xs font-mono text-zinc-500` — ticket IDs, timestamps
- **Labels:** `text-xs uppercase tracking-wider text-zinc-500`
- No Inter. No serif.

## Animation Philosophy

**CSS first.** Use `transition`, `@keyframes`, `animation-delay` cascades for:
- Hover states (`transition-colors duration-150`)
- Focus rings (`transition-shadow`)
- Opacity fades on mount
- Staggered list entry via `calc(var(--i) * 40ms)`
- Scale transforms on active (`active:scale-[0.98]`)

**anime.js** only when CSS can't:
- Spring physics on drag-drop card settle
- Complex orchestrated sequences (bulk action completion)
- Dynamic stagger counts computed at runtime

Durations: 100-250ms max. Easing: `cubic-bezier(0.16, 1, 0.3, 1)` for CSS, `easeOutExpo` for anime.js.

## Anti-Patterns (Banned)

- No emoji in UI. Use Phosphor icons.
- No pure black (`#000`) or pure white (`#fff`).
- No purple/AI aesthetic.
- No sliding panels or drawers.
- No Inter font.
- No generic card-in-card nesting.
- No arbitrary z-index spam.
- No `h-screen` (use `min-h-[100dvh]`).
